Motivation

Why Smart Bike Lights?

Night-time cycling involves rapidly changing lighting — vehicle headlights, streetlamps, reflections — yet most bike lights are fixed-mode.

  • High beam maximises rider visibility on dark paths
  • Low beam reduces glare for oncoming road users
  • Off in daylight — saves battery

We need a system that automatically adapts beam state based on the scene — no manual switching.

System Design

System Architecture

  • ESP32-CAM streams low-res MJPEG over Wi-Fi
  • Raspberry Pi 5 runs CNN inference at 10 Hz
  • Temporal stabilisation — threshold averaging, hold time, anti-flicker logic
  • GPIO drives relay switching headlight modes
  • 3D-printed enclosure protects hardware on the bike

Training

Dataset & Training Pipeline

  • BDD100K driving dataset — 70K+ derived beam labels
  • Rule-based labelling: time-of-day + hazard bounding boxes → off / dim / high
  • Class imbalance: off 56.5%, dim 34.9%, high 8.6%
  • Oversampling factor K=3 for minority class
  • AdamW + cosine schedule + EMA weights
  • 224×224 input, ImageNet normalisation
  • Augmentation: colour jitter, gamma, Gaussian blur

+ 1,979 real bike-cam images for on-bike validation

Conclusion

Key Takeaways

  • Single lightweight CNN outputs beam state (off / dim / high) end-to-end
  • 93.99% accuracy on 18K test images at 49 FPS
  • Policy-derived labels from BDD100K — no manual annotation
  • Temporal stabilisation ensures flicker-free on-bike control
  • Complete edge deployment: ESP32-CAM + RPi 5 + relay + 3D enclosure
